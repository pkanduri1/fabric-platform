**Fix:** Add `'OPERATIONS_MANAGER'` to mock roles in `authApi.ts` lines 95 and 173. Role guard logic stays intact.

**File:** `fabric-ui/src/services/api/authApi.ts`

## #48 — HomePage Dead Buttons + Run-All Backend Endpoint

### Frontend

Button | Action
-------|-------
System Settings | `navigate('/settings')` (or `/configuration` if no settings route)
Run All Jobs | Open source-system picker dialog → confirm → `POST /api/v1/jobs/run-all?sourceSystem=X` → toast with submitted count
Export Configuration | `navigate('/configuration')` with export intent (no backend endpoint in scope)

**Run All Jobs flow:**
1. Click button → dialog opens, lists source systems from `useSourceSystemsState()`
2. User selects one source system
3. Confirmation: "Run all jobs for [SOURCE]?"
4. POST to backend → toast "N jobs submitted for X" or error

**Files:** `fabric-ui/src/pages/HomePage/HomePage.tsx`

### Backend

```
POST /api/v1/jobs/run-all?sourceSystem={sourceSystem}
Role: ROLE_API_EXECUTOR
Response 202:
{
  "sourceSystem": "COLLECTIONS",
  "submittedCount": 5,
  "executionIds": ["EXEC-001", "EXEC-002", ...]
}
```

**Implementation:**
- `JobExecutionApiController` — new `runAll(@RequestParam String sourceSystem, Principal principal)` method
- `JobExecutionApiService.runAllForSource(sourceSystem, actor)` — calls `ManualJobConfigRepository.findBySourceSystemAndStatus(sourceSystem, "ACTIVE")`, invokes `submitJob()` for each, returns summary DTO
- New response DTO: `RunAllJobsResponse { sourceSystem, submittedCount, executionIds }`

**Files:**
- `controller/JobExecutionApiController.java`
- `service/JobExecutionApiService.java`
- `dto/jobexecution/RunAllJobsResponse.java`

## #50 — Audit Trail Returns Real Data

**No new table needed.** The execution row contains all audit-relevant fields.

**Repository:**
```java
List<JobAuditResponse.AuditEntry> findAuditTrail(String executionId);
```
Query `MANUAL_JOB_EXECUTION` by `EXECUTION_ID`. Map into `AuditEntry` records:
- Entry 1: `action=SUBMITTED`, `timestamp=START_TIME`, `actor=EXECUTED_BY`, `correlationId=CORRELATION_ID`
- Entry 2 (if completed/failed): `action=STATUS`, `timestamp=END_TIME`, `actor=SYSTEM`

**Service:** `JobExecutionApiService.getAuditTrail(executionId)` — calls repo, throws `EXECUTION_NOT_FOUND` if not found.

**Controller:** Remove hardcoded `List.of()`, call service method.

**Tests:**
- Unit: mock repo returns entity, assert audit entries mapped correctly
- Integration: submit a job via POST, then GET audit — assert entries non-empty

**Files:**
- `repository/ManualJobExecutionRepository.java`
- `repository/impl/ManualJobExecutionRepositoryImpl.java`
- `service/JobExecutionApiService.java`
- `controller/JobExecutionApiController.java`

## #51 — TransformationConfigController Full CRUD

**Table:** `FIELD_TEMPLATES` (exists, transformation columns added by `phase2-001`).

**Pattern:** Follows CLAUDE.md repository pattern exactly — JdbcTemplate, interface + impl, thin controller.

### Components

1. `TransformationConfigEntity` — maps `FIELD_TEMPLATES` columns (`TEMPLATE_ID`, `TEMPLATE_NAME`, `SOURCE_FIELD`, `TRANSFORMATION_TYPE`, `VALUE`, `TRANSFORMATION_CONFIG`, etc.)
2. `TransformationConfigRepository` interface — `findAll()`, `findById()`, `findBySourceSystem()`, `save()`, `update()`, `softDelete()`
3. `TransformationConfigRepositoryImpl` — `JdbcTemplate` + `RowMapper`
4. `TransformationConfigService` — thin delegation layer
5. `TransformationConfigController` — wired to service, stubs removed

### Endpoints

Method | Path | Role | Response
-------|------|------|--------
GET | `/v1/transformation/configs` | ROLE_USER | 200 list
GET | `/v1/transformation/configs/{id}` | ROLE_USER | 200 or 404
GET | `/v1/transformation/configs/by-system/{source}` | ROLE_USER | 200 list
POST | `/v1/transformation/configs` | ROLE_ADMIN | 201
PUT | `/v1/transformation/configs/{id}` | ROLE_ADMIN | 200 or 404
DELETE | `/v1/transformation/configs/{id}` | ROLE_ADMIN | 204 (soft delete)

### Tests

- Unit: Mockito, one test per endpoint
- Integration: Oracle, `TEST-` prefixed data, `@BeforeEach` seed + `@AfterEach` cleanup

**Files:**
- `entity/TransformationConfigEntity.java` (new)

- `service/TransformationConfigService.java` (new)
